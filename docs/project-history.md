# project-history

Jira 티켓 하나를 기준으로 실제 코드 변경 내역까지 분석해서, 이력서·포트폴리오에 바로 쓸 수 있는 기술 문서를 생성합니다.

## 언제 사용하나요?

- Jira 티켓 단위로 개발한 기능을 이력서용 문서로 정리하고 싶을 때
- 코드 변경 내역, 기술적 도전 포인트, 면접 대비 질문을 한 번에 만들고 싶을 때
- 포트폴리오나 경력 기술서에 구체적인 수치와 클래스명을 담고 싶을 때

## 트리거 문구

```
"RB-623 이력 정리해줘"
"이 티켓 이력서용으로 정리해줘"
"작업 이력 문서화해줘"
"/project-history RB-623"
```

## 실행 흐름

1. **입력 파싱** — Jira 티켓 키 추출 (URL 또는 `RB-623` 형식)
2. **Jira 데이터 수집** — 메인 티켓 + 하위 이슈 + 코멘트 병렬 조회
3. **Git 브랜치 탐색** — 티켓 번호로 관련 브랜치 검색 + 커밋/diff 수집
4. **코드 분석** — Entity → Service → Controller → Repository 우선순위로 핵심 파일 읽기
5. **문서 생성** — 템플릿 기반으로 섹션 채우기
6. **저장** — `{티켓키}-{제목}.md` 파일로 저장

> **핵심 원칙**: 추측이나 과장 없이, Jira와 git에서 확인된 사실만 작성합니다.

## 생성 문서 구조

```
{티켓키}-{제목}.md
├── 기능 개요          ← 무슨 기능인지, 왜 만들었는지 (2-3문장)
├── 티켓별 구현 내용   ← 실제 클래스명·API 엔드포인트 직접 인용
├── 기술적 도전 포인트 ← 코드에서 확인된 비자명한 결정·해결 방식
└── 이력서 작성 포인트
    ├── 한 줄 요약       ← 경력 기술서 '주요 프로젝트'용 1문장
    ├── 주요 업무 bullets ← 3-5개 (동사 명사형 + 구체적 임팩트)
    ├── 사용 기술        ← 코드에서 실제 확인된 기술만
    ├── 면접 대비 Q&A   ← 예상 질문 2-3개와 답변 방향
    └── 주의사항         ← 역효과 날 수 있는 표현·맥락
```
